## English

AI-Diary-Tool is a local desktop tool designed for a personal diary workflow.

It turns your voice notes, transcripts, or manually prepared raw text into a LaTeX diary that can be saved, compiled, and archived directly. It uses the Codex workflow available on your machine instead of Gemini.

### Features

- Paste raw text and generate a complete LaTeX diary
- Import audio files and transcribe them with local `faster-whisper` first
- Preserve a fixed diary template, heading structure, and writing style
- Use local `codex` to generate the diary body
- Optionally compile the result to PDF with `XeLaTeX`
- Store all outputs in the local `outputs/` directory

### Built-in Template Rules

- Document class: `ctexart`
- Compiler: `XeLaTeX`
- Chinese font: `STXingkai`
- English font: `Times New Roman`
- Margins: `2.5cm` left/right, `3cm` top/bottom
- Line spacing: `1.5`
- Header and footer: diary label, date, page number
- A title that summarizes the core theme of the day
- A clearer outline-driven body structure
- Each major point gets its own heading and paragraph block
- `Codex comment:` appears only where it is genuinely useful
- `cite`-style noise is removed automatically
- A smaller closing section titled `Codex?????`

### Run

Double-click:

`run_ai_diary_tool.bat`

Or run from the command line:

```powershell
python app.py
```

### Build a Windows Executable

The project includes:

`build_windows_release.bat`

It generates:

```text
dist/
  AI-Diary-Tool/
    AI-Diary-Tool.exe
    outputs/
    ...
```

The current packaging mode is `--onedir` for better stability with the GUI and Whisper-related dependencies.

### Usage

1. Enter the date
2. Choose simplified or traditional Chinese
3. Set the minimum page length
4. Set the density of `Codex comment` blocks
5. Choose an input mode:
   - Paste a raw transcript
   - Import an audio file and transcribe it first
6. Add any extra requirements if needed
7. Click the generate button
8. The result is saved as `.tex`, with optional `.pdf` output

### Requirements

- Python 3.12+
- A working local `codex`
- A working local `xelatex`
- `ffmpeg` is recommended
- `PyInstaller` is required if you want to rebuild the Windows package

Python dependencies are listed in `requirements.txt`.
